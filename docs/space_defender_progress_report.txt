  ----------------------------------
  ARCHITECTURE PRINCIPLES FOLLOWED
  ----------------------------------

1.  Modular Architecture The game is being designed with separated
    modules such as:

core engine render physics input entity player enemy ui (reserved for UI
systems) utils

Each module has a specific responsibility to keep the project scalable
and maintainable.

2.  Separation of Responsibilities

Game Logic → Entities and systems Rendering → GameCanvas Input →
InputManager Game Loop → GameLoop

This separation prevents large monolithic classes and keeps the system
extensible.

3.  Fixed Timestep Game Loop

The game logic runs at a fixed update rate of:

60 updates per second

Rendering runs independently to maximize smoothness.

  --------------------------
  CORE SYSTEMS IMPLEMENTED
  --------------------------

1.  Game Canvas System

File: core/GameCanvas.java

Purpose: Acts as the primary rendering surface of the game.

Responsibilities: • Drawing all entities • Displaying debug information
(FPS counter) • Managing input listener • Updating entity systems

Key Features: • Extends JPanel • Custom rendering through
paintComponent() • Uses EntityManager for rendering entities • Displays
FPS counter

Rendering Flow:

paintComponent() ↓ EntityManager.render() ↓ draw FPS counter

  ---------------------------------------------------------------------
  2. Game Loop System
  ---------------------------------------------------------------------
  3. Input System

  File: input/InputManager

  Purpose: Captures keyboard input from the user.

  Integration:

  GameCanvas registers the input manager as a KeyListener.

  Responsibilities: • Detect key presses • Pass movement/shooting
  signals to entities • Handle player input commands

  This system will later support:

  movement shooting pause menu navigation
  ---------------------------------------------------------------------

4.  Entity System

Files: entity/EntityManager player/PlayerShip

Purpose: Manage all active game objects.

Entities represent all objects inside the game world.

Examples of entities:

Player Enemies Bullets Powerups Effects

Current Implementation:

EntityManager • Stores entity collection • Updates all entities •
Renders all entities

Update Flow:

GameLoop ↓ GameCanvas.update() ↓ EntityManager.update()

Render Flow:

GameLoop ↓ GameCanvas.repaint() ↓ GameCanvas.paintComponent() ↓
EntityManager.render()

  ---------------------------------------------------------------------
  5. Player Entity
